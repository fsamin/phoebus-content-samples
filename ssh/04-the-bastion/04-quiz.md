# The Bastion Quiz

## [multiple-choice] What is the main purpose of an SSH bastion?

- [ ] To replace SSH with a more secure protocol
- [ ] To speed up SSH connections
- [x] To act as a centralized, mandatory gateway for all SSH access with logging and access control
- [ ] To encrypt files on servers

> **Explanation:** A bastion is a single entry point for all SSH connections. It centralizes authentication, authorization, audit logging, and session recording. Servers only accept connections from the bastion, eliminating direct access.

## [multiple-choice] What is a "protocol break" in The Bastion?

- [ ] The bastion blocks insecure SSH protocols
- [x] The bastion terminates the incoming SSH connection and opens a new one to the target, enabling session inspection and recording
- [ ] The bastion forces users to change protocols
- [ ] The connection is broken and re-established every 5 minutes

> **Explanation:** Unlike ProxyJump (which creates an end-to-end tunnel), The Bastion terminates the first SSH connection and opens a second one. This allows the bastion to record sessions, log activity, and enforce policies — things impossible with a transparent tunnel.

## [multiple-choice] What is an "egress key" in The Bastion?

- [ ] The key you use to authenticate to the bastion
- [x] A key generated by the bastion for your account, used to connect from the bastion to target servers
- [ ] A key used to encrypt audit logs
- [ ] A key shared by all users of a group

> **Explanation:** Each user has egress keys (generated on the bastion) that are used for the second SSH connection (bastion→target). The public part of this key must be installed on target servers. Your personal key (ingress key) is only used for the first connection (you→bastion).

## [short-answer] What is the osh command to list your egress keys?

    selfListEgressKeys

> **Explanation:** `ssh user@bastion --osh selfListEgressKeys` lists the keys the bastion uses on your behalf to connect to target servers. All bastion management commands are prefixed with `--osh`.

## [multiple-choice] What syntax connects to `admin@10.0.1.50` through The Bastion?

- [ ] `ssh admin@10.0.1.50 --via bastion.example.com`
- [ ] `ssh -J bastion.example.com admin@10.0.1.50`
- [x] `ssh user@bastion.example.com -- admin@10.0.1.50`
- [ ] `ssh user@bastion.example.com --target admin@10.0.1.50`

> **Explanation:** The double dash `--` separates the bastion connection from the target specification. Everything after `--` tells the bastion where to connect you.

## [multiple-choice] Why is a bastion critical for compliance (PCI-DSS, ISO 27001)?

- [ ] It encrypts data at rest on all servers
- [ ] It automatically patches servers
- [x] It provides centralized audit trails, session recording, and provable access control
- [ ] It replaces the need for firewalls

> **Explanation:** Compliance frameworks require demonstrating who accessed what, when, and what they did. The Bastion provides all of this: centralized access control, per-session recordings (ttyrec), searchable audit logs, and RBAC — all from a single, auditable point.
